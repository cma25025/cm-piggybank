### P1 — Should fix before beta
- `src/middleware.ts:31` — `await supabase.auth.getUser()` discards the result; Phase 2 route protection needs the user object. Capture it (`const { data: { user } } = await supabase.auth.getUser();`) and stub the redirect now so Phase 2 only adds the policy, not the plumbing.
- `src/middleware.ts:37` — matcher regex does not exclude `/api/*` or `/auth/callback`. Supabase OAuth/PKCE callback needs to set cookies on its own response; running session-refresh middleware over it can race the callback. Add `api|auth/callback` to the negative lookahead before Phase 2.
- `src/lib/distribution.ts:29` — `amount_cents <= 0` rejects zero. A future "round-up the change" or "$0.00 manual adjustment" flow is plausible; consider `< 0` instead, and add an explicit `Number.isFinite` guard so `Infinity` doesn't slip through the integer check on some engines.
- `src/lib/distribution.ts:32` — no validation that individual bps are non-negative integers. `{spend:-100, save:5000, share:5100}` sums to 10_000 and passes, then `Math.floor` of a negative produces a nonsense split. Add `if (spend_bps < 0 || save_bps < 0 || share_bps < 0 || !Number.isInteger(...))` before the sum check.
- `package.json` — no `engines` field and no `.nvmrc`. Phase 1 brings teammates/CI in; pin Node 20.x (matches `@types/node ^20`) now to avoid Drizzle-style "works on my machine" later.
- `package.json:32` — `next ^14.2.0` is fine, but `eslint` and `eslint-config-next` are absent while `"lint": "next lint"` is wired up. `npm run lint` will prompt interactively on first run and fail in CI. Either remove the script or add the deps.
- shadcn install is missing `avatar`, `progress`, `tooltip`, `popover`, `sheet`. Mockup uses circular initial-avatars (`.child-avatar`, `.request-avatar` at mockup:140, 234), bucket fill bars (`.progress` at mockup:191), and mobile drawers. Install before Phase 4 UI work to avoid a mid-phase install churn.

### P2 — Polish
- `src/app/globals.css:1` — Google Fonts loaded via `@import` blocks render. Use `next/font/google` in `layout.tsx` instead; gets self-hosting, preload, and CLS-free swap for free.
- `src/app/layout.tsx:11` — `<html>` is missing `suppressHydrationWarning` even though `next-themes` is installed and `sonner.tsx` calls `useTheme()`. The moment a ThemeProvider lands, hydration will warn.
- `src/lib/utils.ts:9` — `formatCents` hard-codes `en-US` locale while accepting a `currency` arg. Either drop the currency param (YAGNI for v1, USD-only per plan) or also accept a locale. Half an API is worse than no API.
- `tailwind.config.ts:103` — `require("tailwindcss-animate")` in an ESM config works but trips strict ESLint rules. Convert to `import animate from "tailwindcss-animate"` when ESLint lands.
- `src/lib/supabase/{server,client}.ts` — both unconditionally `!`-assert the env vars. A missing `.env.local` produces a confusing runtime stack trace instead of a clear "set NEXT_PUBLIC_SUPABASE_URL" message. Add a one-line guard or a tiny `env.ts` zod schema.
- No `.editorconfig` and no Prettier config. Two-contributor codebase will drift on quote style and trailing commas within a week.
